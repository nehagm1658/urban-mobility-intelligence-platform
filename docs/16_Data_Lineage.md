# 16. Data Lineage

Column-level lineage for every Gold table, source through to output.

## dim_driver / dim_driver_scd2

| Gold column | Silver source | Bronze source | Original source |
|---|---|---|---|
| driver_id | `silver.drivers.driver_id` (cast to int) | `bronze.drivers.driver_id` | `drivers.csv: driver_id` |
| driver_name | `silver.drivers.driver_name` (trimmed) | `bronze.drivers.driver_name` | `drivers.csv: driver_name` |
| city | `silver.drivers.city` (trimmed) | `bronze.drivers.city` | `drivers.csv: city` |
| city_tier | derived: `CITY_TIER_LOOKUP[city]` (Silver enrichment, not from any source) | -- | -- |
| status | `silver.drivers.status` (uppercased) | `bronze.drivers.status` | `drivers.csv: status` |
| rating | `silver.drivers.rating` (cast to double) | `bronze.drivers.rating` | `drivers.csv: rating` |
| rating_bucket (scd2 only) | derived: `ROUND(rating * 2) / 2` (scd_type2.py) | -- | -- |
| version_number, effective_start/end_date, is_current (scd2 only) | generated by `scd_type2.py` on each run, not sourced from any input file | -- | -- |

## fact_trip

| Gold column | Silver source | Bronze source | Original source |
|---|---|---|---|
| trip_id, driver_id, customer_id, vehicle_id | `silver.trips.*` (cast to int) | `bronze.trips.*` | `trips.json` (JSON Lines) |
| fare_amount, distance_km | `silver.trips.*` (cast to double) | `bronze.trips.*` | `trips.json` |
| request_time, drop_time | `silver.trips.*` (cast to timestamp) | `bronze.trips.*` | `trips.json` |
| trip_duration_minutes | derived: `(drop_time - request_time) / 60` (Silver enrichment) | -- | -- |
| city_tier | derived: `CITY_TIER_LOOKUP[city]` | -- | -- |
| payment_mode, payment_status | `silver.payments.*`, left-joined to trips on trip_id in `gold_transformations.py` | `bronze.payments.*` | `payments.csv` |
| hour_of_day, trip_time_bucket | derived in `gold_transformations.py` from `request_time` | -- | -- |

## revenue_summary_mart / cancellation_analytics / demand_supply_analytics / ops_kpi_summary

All four are aggregations of `fact_trip` (joined with `dim_driver` for
`ops_kpi_summary`'s utilization calc, which also pulls `online_hours`
from `silver.driver_shift`, sourced from `driver_shift.csv`) -- see
`pipelines/gold_transformations.py` for the exact `groupBy`/window logic
of each mart. No columns in these marts are sourced directly from
anywhere other than `fact_trip` + `dim_driver` + `silver.driver_shift`.

## recommendations

Not lineage from a single source -- `recommendation_engine.py` reads
multiple Gold marts plus `silver.weather` and `silver.traffic` directly
(see `docs/15_Recommendation_Engine.md`'s rule table for exactly which
mart feeds which rule) and emits a new derived fact
(`metadata.recommendations` / `data/gold/recommendations`) with no
1:1 source column mapping -- every row is the *output of a business rule*
evaluated over one or more upstream tables, not a copied/transformed
column.

## How to trace a specific bad record

Every rejected record retains its full original JSON in
`error.rejected_records.raw_record_json`, tagged with the `batch_id` it
was rejected in. Cross-reference that `batch_id` against
`metadata.pipeline_runs` to see exactly which run, and
`bronze.<entity>` partitioned by that same `batch_id` to see the raw
Bronze row it came from.
